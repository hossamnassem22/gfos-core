# دليل التعامل مع الحوادث

## حالة فشل الخدمة (Service Failure)
1. **الرصد:** التحقق من سجلات `CircuitBreaker`.
2. **الاستجابة:** تنفيذ `RecoveryStrategy`.
3. **التدقيق:** مراجعة `AuditLogs` لمعرفة السبب الجذري (Root Cause Analysis).

## حالة تسريب البيانات (Data Breach)
1. **العزل:** تفعيل `SecretRotator` فوراً لتغيير كافة بيانات الاعتماد.
2. **التحليل:** استخدام `TraceContext` لتتبع رحلة المهاجم داخل النظام.
